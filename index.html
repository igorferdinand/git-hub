Git & GitHub

Um guia simples e direto sobre controle de versão para iniciantes e desenvolvedores.

Guia de Referencia

🗂️ O que e Git?

Git e um sistema de controle de versao distribuido. Ele registra todas as alteracoes feitas nos seus arquivos ao longo do tempo, permitindo que voce volte a versoes anteriores, compare mudancas e colabore com outras pessoas sem sobrescrever o trabalho de ninguem.

☁️ O que e GitHub?

GitHub e uma plataforma online que hospeda repositorios Git na nuvem. Ele facilita a colaboracao entre desenvolvedores, oferecendo ferramentas para revisao de codigo, abertura de issues, pull requests e muito mais — tudo em cima do Git.

Configuracao Inicial

git config --global user.name

Define seu nome de usuario. Aparece em todos os commits que voce fizer. Exemplo: git config --global user.name "Seu Nome"

git config --global user.email

Define seu e-mail. Vincula seus commits ao seu perfil. Exemplo: git config --global user.email "voce@email.com"

Criando e Clonando Repositorios

git init

Inicializa um repositorio Git na pasta atual. Cria a estrutura oculta .git que rastreia as mudancas.

git clone <url>

Clona um repositorio remoto para sua maquina. Baixa todos os arquivos e o historico de commits.

Ciclo Basico de Trabalho

git status

Mostra o estado atual do repositorio: quais arquivos foram modificados, adicionados ou ainda nao estao sendo rastreados.

git add <arquivo>

Adiciona arquivos a area de stage (preparacao para o commit). Use git add . para adicionar tudo de uma vez.

git commit -m "mensagem"

Salva as mudancas em stage como um ponto no historico. A mensagem deve descrever o que foi alterado.

git log

Exibe o historico de commits do repositorio, com autor, data e mensagem de cada um.

git diff

Mostra as diferencas entre os arquivos modificados e a ultima versao commitada.

Trabalhando com Branches

git branch

Lista todas as branches do repositorio. A branch atual aparece com um asterisco.

git branch <nome>

Cria uma nova branch. Branches permitem desenvolver funcionalidades isoladas sem afetar a branch principal.

git checkout <branch>

Muda para outra branch. Use git checkout -b <nome> para criar e ja trocar para ela.

git merge <branch>

Une as alteracoes de outra branch na branch atual, integrando o trabalho desenvolvido separadamente.

git branch -d <nome>

Deleta uma branch local que ja foi mesclada e nao e mais necessaria.

Conectando ao Repositorio Remoto

git remote add origin <url>

Conecta o repositorio local a um remoto (como o GitHub). origin e o nome padrao para o remoto principal.

git push origin <branch>

Envia os commits locais para o repositorio remoto. Na primeira vez, use git push -u origin main.

git pull

Baixa e integra as atualizacoes do repositorio remoto para o seu repositorio local.

git fetch

Baixa atualizacoes do remoto sem aplicar automaticamente. Util para ver o que mudou antes de integrar.

Desfazendo Alteracoes

git restore <arquivo>

Descarta as mudancas de um arquivo ainda nao commitado, voltando ao estado do ultimo commit.

git reset HEAD <arquivo>

Remove o arquivo da area de stage, sem desfazer as modificacoes no arquivo em si.

git revert <hash>

Cria um novo commit que desfaz as mudancas de um commit anterior, preservando o historico.

git stash

Guarda temporariamente as mudancas atuais sem commitar, deixando a area de trabalho limpa. Use git stash pop para restaurar.

Dica — Fluxo padrao de trabalho

O fluxo mais comum no dia a dia e: git pull para atualizar → editar os arquivos → git add . → git commit -m "descricao" → git push. Seguindo esse ciclo voce mantem o repositorio sempre atualizado e com historico organizado.
